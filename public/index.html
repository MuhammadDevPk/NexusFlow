API Integration Overview

NexusFlow provides a high-throughput, non-blocking ingestion endpoint to accept webhook events, transactional triggers, and background jobs. Incoming payloads are instantly validated, checked for duplicate keys, and logged to active Redis Streams before returning an immediate response. Background workers handle execution asynchronously.

GATEWAY ENDPOINT POST http://127.0.0.1:8000/api/events
CONTENT TYPE application/json

Request Body Specification

Parameter Name Data Type Requirement Validation Rules & Description
queue string Required Target processing queue. Must match a queue defined in config.php:
• pdf-generation (Simulated rendering workloads)
• api-sync (External HTTP synchronization threads)
• data-transform (Database/data scrubbing batches)
name string Required Job handler mapping registered in the orchestrator registry:
• pdf.generate (Requires template/order parameters)
• api.sync (Requires target endpoints/payload arrays)
• data.transform (Requires records_count/operations)
idempotency_key string Required A unique client-supplied token (e.g. UUIDv4 or transaction ID). NexusFlow uses this to prevent double-processing and filter duplicate events.
payload object Required Key-value dictionary containing arguments matching the task handler's required format. Must be valid JSON.

HTTP Response Schemas

202 Accepted Job enqueued successfully. 
{
  "status": "queued",
  "job_id": "pk-472098b1-d419-4a94",
  "stream_id": "1784376125089-0"
}
200 OK Duplicate matched (Completed). 
{
  "status": "completed",
  "duplicate": true,
  "message": "Event already processed successfully"
}
409 Conflict Duplicate in-flight execution. 
{
  "error": "Event is already being processed by a worker"
}
400 Bad Request Payload validation mismatch. 
{
  "error": "Missing job name"
}

curl -X POST http://127.0.0.1:8000/api/events \
  -H "Content-Type: application/json" \
  -d '{
    "queue": "pdf-generation",
    "name": "pdf.generate",
    "idempotency_key": "order-inv-99482",
    "payload": {
      "order_id": 99482,
      "template": "invoice_pdf",
      "client_email": "hello@nexusflow.io"
    }
  }'

Node.js / JS Fetch Client

async function dispatchEvent() {
  const url = 'http://127.0.0.1:8000/api/events';
  const payload = {
    queue: 'api-sync',
    name: 'api.sync',
    idempotency_key: `crm-sync-${Date.now()}`,
    payload: {
      endpoint: 'crm/contacts',
      payload: { name: 'Alice Smith', email: 'alice@example.com' }
    }
  };

  const response = await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(payload)
  });

  const result = await response.json();
  console.log(`Status: ${response.status}`, result);
}

PHP HTTP Webhook Client

<?php

$url = 'http://127.0.0.1:8000/api/events';
$data = [
    'queue' => 'data-transform',
    'name' => 'data.transform',
    'idempotency_key' => uniqid('evt_', true),
    'payload' => [
        'records_count' => 150,
        'operation' => 'scrubbing'
    ]
];

$ch = curl_init($url);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
curl_setopt($ch, CURLOPT_HTTPHEADER, ['Content-Type: application/json']);

$response = curl_exec($ch);
$statusCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
curl_close($ch);

echo "Code: $statusCode \nResponse: $response\n";

PHP Direct Redis Ingestion (No HTTP Overhead)

If your application runs in the same environment, you can bypass the HTTP gateway and enqueue directly to the Redis Stream to minimize network latency:

<?php

$redis = new Redis();
$redis->connect('127.0.0.1', 6379);

$id = uniqid('direct_', true);
$jobData = [
    'id' => $id,
    'name' => 'pdf.generate',
    'payload' => json_encode(['order_id' => 8847, 'template' => 'label']),
    'queue' => 'pdf-generation',
    'attempts' => '0',
    'created_at' => (string)microtime(true),
    'last_error' => ''
];

// Write directly to stream
$streamId = $redis->xAdd('stream:pdf-generation', '*', $jobData);
echo "Directly enqueued to stream. Stream ID: $streamId\n";
